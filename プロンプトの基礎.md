# プロンプトの基礎

## はじめに

プロンプトとは、AI言語モデルに対して指示や質問を与えるためのテキスト入力のことです。効果的なプロンプトを作成することで、AIから望ましい回答や出力を得ることができます。

## プロンプトとは

プロンプトは、AIとのコミュニケーションにおける「問いかけ」や「指示」に相当します。単純な質問から、複雑なタスクの実行依頼まで、さまざまな形式があります。

### プロンプトの種類

1. **質問型プロンプト**
   - 「〜とは何ですか？」
   - 「〜について教えてください」

2. **指示型プロンプト**
   - 「〜を作成してください」
   - 「〜を要約してください」

3. **ロールプレイ型プロンプト**
   - 「あなたは専門家です。〜について説明してください」

## 良いプロンプトの書き方

### 1. 明確で具体的に

曖昧な指示ではなく、具体的な要求を明示しましょう。

**悪い例:**
```
文章を書いて
```

**良い例:**
```
ビジネスメールの書き方について、初心者向けに500字程度で説明してください。
件名の付け方、挨拶、本文の構成、締めの言葉の4つのポイントに分けて解説してください。
```

### 2. コンテキストを提供する

背景情報や目的を伝えることで、より適切な回答が得られます。

**例:**
```
私はプログラミング初心者です。Pythonでリストとタプルの違いを、
実際のコード例を交えて分かりやすく説明してください。
```

### 3. 出力形式を指定する

望ましい回答の形式を明示しましょう。

**例:**
```
以下の情報を箇条書きで整理してください:
- 各項目は簡潔に
- 重要度順に並べる
- 3〜5項目程度
```

### 4. ステップバイステップで考えさせる

複雑な問題には、段階的に考えるよう促します。

**例:**
```
以下の問題を段階的に解いてください:
1. まず問題を分析する
2. 解決方法を複数案検討する
3. 最適な方法を選択し、理由を説明する
4. 実装手順を示す
```

## プロンプトの構成要素

効果的なプロンプトは、以下の要素を含むことが多いです:

1. **役割（Role）**: AIに期待する立場や専門性
2. **タスク（Task）**: 実行してほしい具体的な作業
3. **コンテキスト（Context）**: 背景情報や前提条件
4. **形式（Format）**: 出力の形式や構造
5. **制約（Constraints）**: 文字数制限や除外事項

### 例

```
【役割】あなたは経験豊富なマーケティングコンサルタントです。

【タスク】新商品のSNS投稿文を3パターン作成してください。

【コンテキスト】
- 商品: エコフレンドリーな竹製歯ブラシ
- ターゲット: 環境意識の高い20-30代
- 発売日: 来月1日

【形式】
- 各投稿は140字以内
- ハッシュタグを2-3個含める

【制約】
- 過度な宣伝文句は避ける
- 環境への配慮を強調する
```

## よくある間違い

### 1. 曖昧すぎる指示

「良い文章を書いて」では、何が「良い」のか分かりません。

### 2. 情報の過不足

必要な情報が足りないと適切な回答が得られず、逆に情報が多すぎると焦点がぼやけます。

### 3. 一度に複数のタスクを詰め込む

複雑なタスクは分割して、順番に処理する方が効果的です。

## プロンプトの改善プロセス

1. **初期プロンプトを作成**: まずシンプルに書いてみる
2. **結果を評価**: 期待通りの出力か確認
3. **改善点を特定**: 何が足りないか、何が余分か分析
4. **プロンプトを修正**: より具体的に、または構造を整理
5. **再度テスト**: 改善された結果が得られるか確認

## 実践例

### 例1: コード生成

```
Pythonで、CSVファイルを読み込んで特定の列の平均値を計算する関数を作成してください。

要件:
- 関数名は calculate_column_average
- 引数: ファイルパス、列名
- エラーハンドリングを含める
- docstringを付ける
- 使用例も示してください
```

### 例2: 文章の添削

```
以下のビジネスメールを添削してください。
改善点を「修正前」「修正後」「理由」の形式で示してください。

[メール本文]
お世話になります。
先日の件ですが、ちょっと問題が発生しました。
至急対応お願いします。
```

## まとめ

効果的なプロンプトを作成するには:

- **明確性**: 何を求めているか明確に
- **具体性**: 詳細な情報と例を提供
- **構造化**: 整理された形式で指示
- **反復改善**: 結果を見ながら改善

プロンプトエンジニアリングは、実践を通じて上達するスキルです。さまざまなプロンプトを試し、結果を観察しながら、自分なりの効果的なパターンを見つけていきましょう。
